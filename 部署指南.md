# 🚀 全自动部署指南

## 第一步：获取API密钥（15分钟）

### 1.1 DeepSeek API（必需）

1. 访问 https://platform.deepseek.com
2. 注册账号并登录
3. 进入「API Keys」页面
4. 点击「创建API Key」
5. 复制生成的Key（格式：sk-xxxxxxxx）

**费用**：¥1-2/百万tokens，生成一条脚本约¥0.01

### 1.2 飞书应用（必需）

1. 访问 https://open.feishu.cn/app
2. 点击「创建企业自建应用」
3. 填写应用名称："短视频自动化"
4. 进入应用管理页面
5. 复制 **App ID** 和 **App Secret**
6. 进入「权限管理」，添加以下权限：
   - `bitable:record`
   - `bitable:view`
   - `im:chat:send`

### 1.3 飞书表格（必需）

1. 在飞书创建多维表格
2. 添加字段（见下方字段列表）
3. 点击右上角「...」→「复制链接」
4. 从链接中提取：
   - `App Token`：base/后面的字符串
   - `Table ID`：table=后面的字符串

**字段列表**：
| 字段名 | 类型 |
|--------|------|
| 标题 | 文本 |
| 热度值 | 数字 |
| 来源平台 | 文本 |
| 链接 | 文本 |
| 匹配关键词 | 文本 |
| 匹配分数 | 数字 |
| 抓取时间 | 日期时间 |
| 处理状态 | 单选：待审核/脚本已生成/已发布/已忽略 |
| 脚本标题 | 文本 |
| 脚本内容 | 文本 |
| 标签 | 文本 |
| 生成时间 | 日期时间 |

### 1.4 飞书群机器人（必需）

1. 在飞书群聊中点击「设置」
2. 选择「群机器人」→「添加机器人」
3. 选择「自定义机器人」
4. 复制 **Webhook地址**

---

## 第二步：Fork项目并配置（10分钟）

### 2.1 Fork项目

1. 访问项目页面
2. 点击右上角「Fork」按钮
3. 等待Fork完成

### 2.2 配置Secrets

1. 进入Fork后的项目
2. 点击「Settings」→「Secrets and variables」→「Actions」
3. 点击「New repository secret」，依次添加：

| Secret名称 | 值 |
|------------|-----|
| `DEEPSEEK_API_KEY` | 从1.1获取 |
| `FEISHU_APP_ID` | 从1.2获取 |
| `FEISHU_APP_SECRET` | 从1.2获取 |
| `FEISHU_APP_TOKEN` | 从1.3获取 |
| `FEISHU_TABLE_ID` | 从1.3获取 |
| `FEISHU_WEBHOOK` | 从1.4获取 |

---

## 第三步：首次运行测试（10分钟）

### 3.1 手动触发工作流

1. 进入项目「Actions」标签页
2. 点击「Auto Short Video Pipeline」
3. 点击「Run workflow」
4. 选择 `job: crawl-hotspots`
5. 点击「Run workflow」

### 3.2 查看运行结果

1. 等待工作流运行完成（约1-2分钟）
2. 点击运行记录查看日志
3. 检查飞书表格是否有数据写入
4. 检查飞书群是否收到通知

### 3.3 测试脚本生成

1. 再次点击「Run workflow」
2. 选择 `job: generate-scripts`
3. 运行完成后检查飞书表格
4. 查看"脚本内容"字段是否有内容

---

## 第四步：启用自动化（5分钟）

工作流已配置定时任务，无需额外操作：

| 时间 | 任务 |
|------|------|
| 每天6:00 | 自动抓取热点 |
| 每天7:00 | 自动生成脚本 |
| 每天8:00 | 自动发送日报 |

如需修改时间，编辑 `.github/workflows/auto-short-video.yml` 文件中的 `schedule` 部分。

## 故障排查

### 问题1：工作流运行失败

**检查步骤**：
1. 进入Actions页面查看错误日志
2. 检查Secrets是否配置正确
3. 检查API Key是否有效

### 问题2：飞书表格无数据

**检查步骤**：
1. 检查App Token和Table ID是否正确
2. 检查飞书应用是否有表格权限
3. 检查表格字段名称是否匹配

### 问题3：脚本生成失败

**检查步骤**：
1. 检查DeepSeek API Key是否有效
2. 检查API余额是否充足
3. 检查网络连接
